复杂任务的单一工作流

把模糊目标，变成已验证结果。

AI 负责查清、想透、拆好、做成、验真；你只在价值、取舍、验收和授权时决定。

七个动作不是七场仪式。已有证据就快速通过；简单事直接做。每一步只增加结果、降低风险、减少重复。

为什么需要 workflow

复杂工作最怕的，不是难，而是一路都在猜。

目标没说清就开工，方案没比较就押注，任务没验收就并行，代码写完就宣布交付。workflow 用三段七动作，把每一次猜测变成事实、决策或证据。

计划不确定 → 承诺
需求澄清选定方案拆成任务
执行承诺 → 证据
执行任务验收交付
复盘结果 → 能力
提炼经验回灌改进

证据是快速通过的凭证，不是额外仪式。轻任务绕过完整流程，关键价值门、验收门和授权门不降级。

01计划不确定 → 承诺

保持稳定 · 需求澄清

不先做，先证明值得做。

这一步只回答：为什么做，怎样才算成？

痛点

需求看似明确，价值、边界和成功标准却仍靠猜。

解决方案

先查事实，再按决策树成批问清关键判断。

最佳实践

独立问题编号批问；承重分支没关就不出需求卡。

谁做什么

查事实、排问题、追关键分支。
你
按编号确认取舍与授权。
产物
目标、成功信号与非范围。
证据
关键判断有回答或明确处置。

真实变化“做个 AI 助手”变成“权限内问答，确认后拆单；不自动执行”。

- 结论：只做权限内问答与确认后拆单。

07复盘结果 → 能力

保持稳定 · 回灌改进

发现好改进，先让你决定。

这一步只回答：什么值得提案，应该在哪里改？

痛点

按次数回灌会漏掉高价值洞察，也会升级低价值噪音。

解决方案

AI 先做最小提案，确认后交给正确项目完成。

最佳实践

只提小改动、大价值；不值得就不改。

谁做什么
AI
识别痛点、做减法、推荐落点。
你
确认需求、方案与计划。
产物
已确认提案，或明确不改。
证据
目标位置完成改动并重新验明。

真实变化发现固定次数门压制高价值洞察；用户确认后在目标项目追加计划并验明。

## 已确认回灌提案
- 痛点：固定次数门会错过单次高价值洞察。
- 推断需求：让 AI 做价值判断，再由用户决定。
- 最小改造：替换次数门，复用现有计划与交接胶囊。
- 预期价值：减少低价值规则、重复沟通和无意义上下文。
- 验收：目标位置完成改动并返回 fresh 证据。

不确定，不等于追问

先判断未知属于哪一路。

workflow 不把调查工作转嫁给用户，也不把需要决策的取舍伪装成技术问题。四路未知处理完，都回到当前业务动作。

01

事实可查

缺少客观事实

查项目、数据、工具、配置和可信来源，不把可发现事实问给用户事实有来源，或证实不可得
02

取舍待定

缺少决策 owner 的选择

给推荐、代价和默认假设，必要时 Grill 用户决策 owner 作出选择
03

假设待验

讨论无法证明真假

定义最小实验、原型、小样或可失败验收假设被证实、证伪或明确保留风险
04

外部待解

依赖范围外事实或权力

写清 blocker、外部 owner、解锁条件和安全等待方式获得授权/事实，或调整范围

复杂决策才启用

先看见整片森林，再选路线。

workflow 不预设专家名单。它先判断“把这件事做到 90 分，谁最能改变结论”，再邀请 3–5 个关键视角，用同一份事实完成独立判断和相互挑战。

简单、低风险、容易回退的任务不会为了“专家感”强行开会。

01

独立发散

每个专家分别勾勒终局、依赖、盲区和会改变判断的新证据。

02

交叉质询

挑战未经证明的假设、局部最优和被忽略的角色或生命周期。

03

主持收敛

合成完整森林图景，比较 2–3 条路线，给出推荐、备选和删除理由。

最后一步

安装 workflow。

只安装一个 skill。最省事的方式，是把左侧指令完整复制给你的 Agent；它会自动识别自己的 skills 目录并在安装后验证。

复制给你的 Agent

请安装 GitHub 项目 https://github.com/qzl0215/workflow。先把仓库克隆到临时目录，再根据你当前 Agent 的配置确认 skills 父目录，不要猜固定路径；运行 python3 scripts/install.py install --target "<skills父目录>"。如果已经安装，则使用 update。最后运行同一脚本的 check，只有验证通过后才能告诉我安装完成。不要安装任何其他 skill，也不要覆盖没有备份的旧版本。

或在终端安装

  1. git clone --depth 1 https://github.com/qzl0215/workflow.git
  2. cd workflow
  3. python3 scripts/install.py install
  4. python3 scripts/install.py check --target "<skills父目录>"
