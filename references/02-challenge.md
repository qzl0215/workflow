# 02 fallback｜独立视角挑战

只有当前关注点没有用户指定或宿主原生独立复核能力，且额外视角可能改变关键决定时才读取。参与规模只由当前风险决定，不预设角色表或人数。

## 最小输入

所有视角共享同一份短简报：目标、事实、约束、验收、候选路线、关键未知，以及最需要挑战的判断。每个视角先独立判断，不先看其他结论。

## 最小方法

1. 按当前风险动态选择真正相关的视角，只增加能提供不同证据或反例的参与者，不凑人数。
2. 各视角分别检查端到端关系、遗漏角色与状态、依赖、反例、难逆风险，以及局部最优对整体的伤害。
3. 交叉挑战未经证明的假设，并做删除测试：删掉某项是否仍能达到用户结果。
4. 总协调者区分事实、推断与观点，按证据合成路线比较和推荐，不以多数票代替判断。
5. 新增视角已不再改变范围、路线或主要风险时立即停止。无独立 Agent 时可由同一模型显式切换职责，不冒充外部评审。

## 输出与返回

只输出会改变决定的独立贡献、反例、冲突、推荐和删除项；综合结论写入 `findings.md`，不保存会议流水。完成后返回 02 主协议。
