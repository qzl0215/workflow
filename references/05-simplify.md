# 05 fallback｜代码整理

只有实际 diff 已出现重复实现、不必要复杂度、维护性或明显效率风险，或用户明确要求 simplify，且当前关注点没有用户指定或宿主原生代码整理能力时才读取。没有具体信号时读取数为 0；本文件自包含，不要求外部能力。

## 最小输入

业务验收、实际 diff、相关代码 owner、项目约定、测试与性能证据。只审当前变更及必要相邻面，不把整个仓库扩成重构项目。

## 最小方法

1. **复用**：查已有 helper、组件、类型、配置和模式，删除重复实现与第二真源。
2. **删除**：移除不改变用户结果或关键风险的代码、抽象、兼容层和配置；能删不加。
3. **合并与清晰**：减少重复分支、过度拆层、宽接口、隐式状态和没有现实需求的通用化。
4. **效率**：检查重复工作、无界扫描、额外网络/磁盘往返、明显 N+1、过大上下文和无收益缓存/并发。
5. 只修证据确认且在当前范围内的问题；更大重构回 03 成为独立 Plan，不能在验收中偷扩范围。
6. 修改后 fresh 重跑受影响验证，再按实际 diff 复核；代码更短不能替代行为正确。

## 输出与返回

输出已复用、删除或合并的内容、保留理由、效率检查、实际修改和 fresh 结果。没有可整理项可以明确 no-op；处理完后返回 05 主协议。
