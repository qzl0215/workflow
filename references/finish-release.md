# Finish、交付与发布安全

## 触发

实现和 fresh verification 已完成，准备交付文件、总结结果、处理 Git、发布、部署或清理。

## 收尾协议

1. 对照范围说明完成了什么、未做什么、验证证据、残余风险和用户可见变化。
2. 面向非技术读者先给业务结果、是否可放心使用和下一步；技术细节只保留复核所需部分，避免把 changelog 当交付摘要。
3. 检查工作区、生成物、版本、变更说明、迁移/回滚和敏感信息；不混入无关改动。
4. 将动作分级：本地可逆文件修改可按任务执行；commit、push、merge、deploy、delete、公开 release 等必须获得用户对目标和范围的明确授权。
5. 不 force push、不绕过保护、不把凭据写入文件或日志；目标 remote、branch、环境和候选 digest 必须先只读确认。
6. 失败时停止后续写操作，保留可恢复状态和证据；不要把本地完成描述成已发布。

## 无 Git / 无平台能力

仍可交付文件、diff、验证命令、安装说明和明确的人工下一步；不得伪造 commit、PR、URL、部署或线上状态。

## 出口

最终状态只能是：本地已验证、已提交、已推送、已合并、已部署或被明确 blocker 阻断；每一级都需对应证据和授权。
