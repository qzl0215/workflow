# Context Handoff

## 触发

上下文将满、跨会话、切换阶段/执行者，或当前任务需要比已有摘要更深的证据。

## L0–L4 加载

- **L0 定位**：项目、任务目录、当前阶段、活跃 Plan/Task。
- **L1 快照**：task plan 的 Current Snapshot 与 progress 的 handoff checkpoint。
- **L2 当前 Task**：验收、依赖、文件域、相关 findings 和最新证据。
- **L3 目标证据**：只加载能回答已记录缺口的代码、日志、reference 或历史决策。
- **L4 全历史**：仅在一致性争议、审计或 L3 仍不足时使用，并记录原因。

## Task capsule

交接至少包含：目标、当前阶段、输入/证据路径、允许与禁止文件、已完成动作、失败与排除假设、验收、验证命令、权限、blocker 和下一步。

新执行者先验证 capsule 中最易过期的事实；缺上下文时返回具体请求，不自行加载整个项目。摘要与文件冲突时，以文件真源和 fresh 检查为准并 fail closed。

## 出口

另一位执行者无需读取整段聊天即可安全继续当前 Task，同时知道哪些历史不应加载。
