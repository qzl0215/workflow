# Context Discovery

## 何时读取

仅当现有上下文不足以判断项目事实、可用工具、运行环境或用户已表达偏好时读取。

## 协议

1. 先查当前对话、项目规则、文件树、配置、历史计划和可用工具。
2. 把信息分为：已证实事实、带来源的推断、会改变方向的未知、暂不需要知道的内容。
3. 能通过只读检查获得的事实不问用户；不要请求可自行发现的路径或命令。
4. 只有主观取舍、缺失凭据、外部授权或高成本未知才向用户提问，并说明它会改变什么。
5. 记录能力边界：执行、浏览器、图像、subagent、memory、Git、网络和外部系统是否可用。

## 出口

当前阶段所需事实已足够，或明确停在一个可行动 blocker。后续只携带最小证据摘要，不复制整份历史。
