# Design System

## 触发

多个界面任务需要共享设计决策，或现有设计真源缺失/冲突导致后续实现会重复猜测。单页小改不强制创建 DESIGN.md。

## 最小协议

1. 先审计现有 tokens、CSS、组件、品牌规则和页面模式，确定真实代码 owner。
2. 优先补现有设计真源；只有 UI/品牌范围存在、跨任务价值明确且没有 owner 时，才创建项目根 `DESIGN.md`。非 UI 任务不得创建空 DESIGN.md。
3. DESIGN.md 只保存稳定决策：atmosphere/density、语义色彩角色、字体层级、间距/布局、组件与页面模式、响应式、motion、完整状态和 do/don't。
4. 代码 tokens 和组件实现仍是数值/行为细节真源；文档链接它们，不复制整套值。
5. 每条规则必须可落实，删除“高级、现代、干净”等无约束形容词。

## 出口

实现者能找到唯一设计 owner，并能把 Experience Contract 映射到现有或新增 token、组件和页面模式；否则把缺口退回设计阶段。
