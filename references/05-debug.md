# 05 条件｜根因诊断

## 触发

出现 bug、测试/构建失败、性能或行为异常、空结果、flaky、真实验收失败，或同一路径连续失败。单一 typo 可最小修复并相称验证，不扩成大型诊断。

## 最小上下文与角色

Debug owner 只接收预期、实际、最小复现、影响面、错误/日志、输入、环境、最近变更、source fingerprint 和已排除假设；不默认读完整 Request 或无关模块。

## 做法

1. 写清预期、实际、影响面和最小复现；不能复现先增加可观测性。
2. 分开事实与假设，沿数据/控制流向上找到第一个错误状态，不在末端症状堆补丁。
3. 一次验证一个可证伪假设，使用最小实验、对照和条件等待；不加固定 sleep 掩盖竞态。
4. 修复根因的最小表面，在正确边界补回归；避免无关重构。
5. 每次失败都必须增加新证据或切换本质不同的观察点；同类失败反复发生时反转核心假设并扩大针对性证据，禁止原样重试。
6. 先重跑复现，再跑相邻回归和上层验收；flaky 必须证明等待条件和稳定性。

## 输出与返回

输出复现、事实、假设、实验、根因、最小修复、fresh 结果和残余风险。根因能解释现象且复现转绿后返回 05 主协议；事实表明是需求/设计/计划缺口时路由 02/03/04；危险诊断或新权限保持 blocked。
