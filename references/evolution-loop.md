# workflow Evolution Loop

## 触发

修改本包的入口、reference、template、script、test 或生成文档，或复杂任务复盘得到可跨项目复用的改进。

## 变更协议

1. 从真实失败或重复成本写最小问题陈述；单次偶发现象先记录，不急于扩机制。
2. 判断唯一 owner。能改现有文件就不新增；没有独立触发的内容不得新建 reference。
3. 先定义可失败的验收或测试，再做最小变更；保持根入口只负责判级、硬门和路由。
4. 路由/状态变化必须同步 doctor、行为测试和 visual map 生成源；模板/schema 变化同步所有 writer/reader。
5. 检查单入口、外部依赖、孤儿、重复协议、平台硬编码、包大小和渐进加载预算。
6. 运行受影响测试和 fresh full gate；记录 source fingerprint、命令与结果。
7. 只有稳定跨任务收益高于维护成本时才保留机制，否则删除或合并。

## 出口

变更有唯一 owner、可机械验证、未制造第二真源或外部依赖，生成文档与正式真源一致。
