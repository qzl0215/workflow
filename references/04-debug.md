# 04 fallback｜根因诊断

只有出现可观察的 bug、测试/构建失败、性能或行为异常、空结果或不稳定结果，且没有用户指定或宿主原生调试能力时才读取。没有真实失败时读取数必须为 0。

## 最小输入

预期、实际、影响面、最小复现、错误或日志、输入、环境、最近变更、source fingerprint 和已排除假设。不要默认读取完整 Request 或无关模块。

## 最小方法

1. 写清预期、实际、影响面和最小复现；无法复现时先增加最小可观测性。
2. 分开事实与假设，沿数据或控制流寻找第一个错误状态，不在末端症状堆补丁。
3. 一次验证一个可证伪假设，使用最小实验、对照和条件等待；不以固定 sleep 掩盖竞态。
4. 修复根因的最小表面，并在正确边界补回归；避免无关重构。
5. 每次失败必须增加新证据或切换本质不同的观察点；禁止原样重试。
6. 先重跑复现，再跑受影响回归和上层验收；不稳定问题必须证明等待条件与稳定性。

## 输出与返回

输出复现、事实、假设、实验、根因、最小修复、fresh 结果和残余风险。根因解释现象且复现转绿后返回 04 主协议；事实表明是需求、方案或计划缺口时分别返回 01、02、03；危险诊断或新权限保持等待。
